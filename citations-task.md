# Задача: цитаты, источники и режим «не знаю» (анти-галлюцинации)

> **Статус: РЕАЛИЗОВАНО (2026-07-05).** Работа целиком в границах `webchat`; контракт
> `../rag /search` не менялся. Итоги прогона — в `eval/REPORT.md`.

## Цель
Ответ с RAG обязан сопровождаться **источниками** и **дословными цитатами** из найденных
чанков, а при слабом контексте ассистент должен честно сказать **«не знаю»** и попросить
уточнение — вместо тихого ответа из собственных знаний модели (анти-галлюцинации).

## Что было не так (до)
- Цитаты не отдавались клиенту: текст чанка уходил в system-prompt, но структурированной
  цитаты в ответе не было.
- Режим «не знаю» был **инвертирован**: когда фильтр отбрасывал все чанки, backend молча
  отвечал из знаний модели — противоположно требованию.
- Ничто не проверяло, что ответ реально опирается на цитаты.
- `eval/compare.py` мерил только retrieval-hit; путь «не знаю» не проверялся.

## Решения (согласованы с пользователем)
1. **Цитаты** — модель возвращает JSON `{"answer","used":[{"id","quote"}]}`; backend
   валидирует, что `quote` — дословная подстрока текста чанка, иначе отбрасывает.
2. **«Не знаю»** — порог на `rerank_score` (или cosine `score`); если ни один чанк не
   прошёл — `ABSTAIN_REPLY` без вызова генерации, без отката на знания модели. Порог в
   обоих режимах (plain и improved).
3. **Идентификатор источника** — `file :: section` + номер `[i]`. Контракт `../rag` не
   трогали (`chunk_id` не вводили).
4. **Проверка** — LLM-судья DeepSeek (grounded) + авто-проверки + набор off-topic.

## Изменения
- `backend/main.py`:
  - `build_system_prompt` — контекст с нумерацией `[1..n]` + инструкция JSON-вывода.
  - `call_deepseek(..., response_format)` — JSON-режим DeepSeek для cited-генерации.
  - `parse_grounded_reply` — парсинг + substring-валидация цитат (`quotesDropped`).
  - `fetch_rag_context` — порог-гейт; возвращает чанки + `ragMeta.abstained`.
  - `chat` — abstain до генерации; `Source.quote`; `ragMeta` (`abstained`,
    `quotesDropped`, `modelAbstained`).
- `frontend/`: `types.ts` (`quote?`), `App.tsx` (рендер цитаты), `App.css` (`.quote`).
- `eval/`: `compare.py` (cited-прогон + авто-проверки + судья + off-topic), `questions.json`
  (+5 off-topic, id 13–17), генерирует `REPORT.md`.

## Verify (как проверить)
1. RAG-сервис: `curl 127.0.0.1:8100/health` → `{"ok":true}`.
2. Backend (тест-инстанс, порт свободный, ключ в `.env`):
   - in-domain `useRag=true` → `sources` с непустыми `quote`, в тексте ссылки `[i]`,
     `quotesDropped=0`, `abstained=false`.
   - off-topic `useRag=true` (оба режима) → `reply`=«не знаю…», `sources=[]`,
     `ragMeta.abstained=true`.
3. `BACKEND_URL=... python eval/compare.py` → `REPORT.md`:
   источники и цитаты присутствуют у всех in-domain, судья `grounded`, off-topic → abstain.
4. Frontend `npm run build` — зелёный; в UI под ответом видны источники и цитаты.
