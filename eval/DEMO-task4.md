# Задание 4 — цитаты, источники и анти-галлюцинации: демонстрация

Задание 4: ответ с RAG обязан возвращать **источники** (файл + секция) и **дословные
цитаты** из найденных чанков, а при слабом контексте — сказать **«не знаю»** и
попросить уточнение.

## Подготовка

- Открыть [jorchik.com](https://jorchik.com), **жёсткая перезагрузка Ctrl+Shift+R**
  (чтобы подгрузился фронт с отображением цитат), очистить историю чата.
- Под полем ввода два чекбокса: **«RAG»** и **«Улучшенный RAG»** — в сценариях 1–2
  включены оба.

---

## 1. Обязательные источники и цитаты

**Опции:** ☑ RAG · ☑ Улучшенный RAG

**Вопрос:**

```
What fields does the ErrorMessage data class in JetNews contain?
```

**Результат:** ответ `id: Long`, `messageId: @StringRes Int`, а под ним блок
**«Источники»** с файлом `…/ErrorMessage.kt` и **дословной цитатой в рамке**:
`data class ErrorMessage(val id: Long, @StringRes val messageId: Int)`.

**Почему:** с включённым RAG генерация идёт в JSON-режиме DeepSeek — модель обязана
вернуть ответ и для каждого источника дословный фрагмент. Backend проверяет, что цитата
реально является подстрокой текста чанка (`parse_grounded_reply`); выдуманные цитаты
отбрасываются. Источник = `файл :: секция`.

Ещё пример для второго дубля:

```
What copyright year is stated in the top-level (root) README license header of the Jetpack Compose Samples repository?
```

**Результат:** «2024» + источник `README.md` + цитата `Copyright 2024 The Android Open Source Project`.

---

## 2. Режим «не знаю» при слабом контексте

**Опции:** ☑ RAG · ☑ Улучшенный RAG

**Вопрос не по базе:**

```
What is the capital of Australia?
```

**Результат:** «**Не знаю: в найденных источниках нет ответа на этот вопрос. Уточните
или переформулируйте вопрос.**» Блок «Источники» пустой.

**Почему:** если релевантность всех чанков ниже порога (`rerank_score < −6`) — backend
отдаёт заранее заданный ответ «не знаю» **без вызова генерации**, не откатываясь на
знания модели. То же происходит, если после валидации не осталось ни одной цитаты.

---

## 3. Контроль: «не знаю» — это правило, а не отказ модели

**Опции:** ☐ RAG (оба выключены)

**Тот же вопрос:**

```
What is the capital of Australia?
```

**Результат:** обычный ответ `Canberra`, без источников.

**Почему:** без RAG порог не участвует, модель отвечает из своих знаний. Контраст с п.2
показывает, что «не знаю» — следствие правила о пороге, а не неспособности модели.

## Проверка на наборе вопросов (терминал)

Backend и RAG-сервис подняты.

```sh
cd ~/repos/webchat/eval && python3 compare.py
```

**Результат:** сводка + `eval/REPORT.md`. Для каждого доменного вопроса проверяется
наличие источников, наличие дословных цитат и соответствие смысла ответа цитатам
(LLM-судья DeepSeek); для off-topic — срабатывание «не знаю». Типичный итог:

```
источники 12/12 · цитаты 12/12 · grounded 10–12/12 · off-topic abstain 5/5
```

**Источники и цитаты — 12/12:** все доменные вопросы возвращают источник с дословной
цитатой. Помогает авто-повтор генерации: если модель в первый раз перефразировала
цитату (валидация её отбросила), backend один раз переспрашивает с более жёстким
требованием копировать дословно — это срезает ложные «не знаю».

**grounded 10–12/12** — это отдельная проверка «смысл ответа следует из цитат»
LLM-судьёй DeepSeek; судья недетерминирован, поэтому счётчик слегка плавает. Ключевое
для Задания 4 (обязательные источники и цитаты) — 12/12.
