# Задание 5 — мини-чат с RAG + память задачи: демонстрация

Мини-чат хранит историю, на каждый вопрос ищет контекст через RAG, отвечает с
источниками, и ведёт **структурированную «память задачи»** (цель / ограничения /
термины / уточнённое). Ниже — что показать в веб-чате и как проверить набором.

## Подготовка

- Открыть [jorchik.com](https://jorchik.com), **Ctrl+Shift+R**, нажать «Очистить».
- Оба чекбокса (**RAG** и **Улучшенный RAG**) включены по умолчанию — RAG работает
  на каждый вопрос, rewrite раскрывает местоимения в multi-turn.
- Под чатом после первого ответа появляется панель **«🧠 Состояние задачи»**:
  Цель · Ограничения · Термины · Уточнено.

## Требование задания → где видно

| Требование | Где в UI |
|---|---|
| хранит историю диалога | сообщения копятся; перезагрузка страницы их сохраняет (localStorage) |
| на каждый вопрос ищет контекст через RAG | блок «Источники» под каждым ответом |
| отвечает с учётом найденного + всегда источники | ответ ссылается на `[i]`, ниже — файл + дословная цитата |
| память: цель диалога | панель → **Цель** |
| память: ограничения / термины | панель → **Ограничения** / **Термины** |
| память: что уточнено | панель → **Уточнено** |
| не теряет цель | Цель держится на всех ходах |

## Сценарий в веб-чате (вести подряд, один диалог)

**1.**
```
In Jetchat, which class holds the conversation UI state?
```
→ ответ + **Источники** (`ConversationUiState.kt` + цитата). В панели появилась
**Цель**, в **Термины** — `ConversationUiState`.

**2.**
```
What constructor parameters does that class take?
```
→ «that class» раскрылось (см. `🔎 поисковый запрос`), ответ + источник. **Цель та же.**

**3.** (задаём ограничение/уточнение)
```
Keep answers concise and focus on the data model, not UI rendering.
```
→ в панели обновляются **Ограничения** / **Уточнено** (модель зафиксировала правило).

**4.**
```
Which UI tests cover the conversation screen in the androidTest suite?
```
→ ответ + источник (`ConversationTest.kt`). Термины пополняются (`ConversationTest`).

**5.** (проверка памяти — recap)
```
Подведи итог, что мы обсудили про состояние беседы в Jetchat.
```
→ ассистент отвечает **из памяти задачи и истории** (без блока «Источники», но и
**не** «не знаю»). Цель и накопленные термины по-прежнему в панели.

5 ходов = 10 сообщений (в диапазоне «10–15»); можно продолжить своими вопросами по
Jetchat, цель держится весь диалог — это и есть «не теряет цель и продолжает выдавать
ответы с источниками». Более длинную формальную проверку (2×7 ходов) даёт `task5_memory.py` ниже.

## Проверка истории (перезагрузка)

После нескольких ходов нажать **F5**: диалог на месте (localStorage). Панель памяти
появится снова после следующего сообщения — «память задачи» хранится на backend по
`session_id`, а не в браузере.

## Формальная проверка: 2 сценария (терминал)

Backend и RAG-сервис подняты.

```sh
cd ~/repos/webchat/eval && ../backend/.venv/bin/python task5_memory.py
```

**Результат:** консоль + `eval/REPORT-task5.md`. Прогоняет 2 сценария по 7 ходов
(≈14 сообщений) в improved-режиме и проверяет инварианты:
1. `task_state.goal` не теряется по ходу диалога;
2. обычный ответ несёт источники; recap-вопрос отвечается из памяти (не abstain).

Ожидаемо: **✅ оба инварианта держатся в обоих сценариях.**

## Примечание про «не знаю»

Если отдельный ход выдаёт «не знаю» — это честный анти-галлюцинационный отказ
(модель не привела дословную цитату, хотя контекст найден); недетерминированно, ~1–3
на 7 ходов. Демонстрируемые инварианты от этого не страдают: цель держится, память
копится, recap отвечается, большинство ходов — с источниками. При желании ход можно
переформулировать и переспросить.
